1. Acceso al sistema

1.1 Página de Login

Acceso desde login.php. Campos obligatorios:

  • Tipo de documento: V, E o J
  • Cédula: solo números (máx. 8 dígitos)
  • Contraseña
Figura: Login.

Enlaces disponibles: ¿Olvidó su contraseña? y Registrarse.

2. Registro de usuario y recuperación de contraseña

2.1 Registro de nuevo solicitante

Formulario en registro_usuario.php con campos: cédula, nombre, gerencia, contraseña y confirmar contraseña.

Figura: Registro de usuario.

2.2 Recuperación de clave

Flujo: `gestion_clave.php` -> `procesar_recuperacion.php` -> `nueva_clave.php` -> `guardar_clave.php`.

3. Panel de Solicitante

Ubicado en index_solicitante.php. Desde aquí puede crear solicitudes o consultar sus tickets.

Figura: Panel de Solicitante.

4. Registro de nueva solicitud

Página registro.php. Pasos:

  1. Seleccionar área del problema.
  2. Seleccionar tipo y marca (se cargan vía AJAX).
  3. Escribir descripción y adjuntar archivos (si aplica).
  4. Presionar Enviar para crear la solicitud.
Figura: Crear nueva solicitud.

5. Consulta de tickets

Página ver_tickets.php. La tabla incluye columnas: ID, descripción, solicitante, técnico, estado, fecha, acciones.

Figura: Listado de tickets y filtros.

Acciones

  • ACEPTAR: asigna el ticket al especialista/técnico.
  • ASIGNAR: reasignación a otro técnico.
  • VER: ver detalle y comentarios.
  • CERRAR: cerrar con comentario final.

6. Roles y funciones (Especialista / Técnico / Jefe)

Panel de especialista: views/home_especialista.php. Funcionalidades: Gestión de tickets, Control de personal, Reportes y auditoría.

Si el manual se sirve desde el servidor (por ejemplo http://localhost/sistema_fondas/), puedes ver el panel en vivo a continuación. Si estás abriendo el HTML por fichero, usa el enlace alternativo.
Para ver el manual completo en el navegador, abre este enlace en tu navegador web: http://localhost/sistema_fondas/manual_usuario_fondas.html. Desde esa página podrás descargar el manual como PDF usando el botón de impresión del navegador.
Descargar como PDF

Notificaciones

Polling cada 3s a ajax/check_new_tickets.php. Si hay nuevos tickets se muestra alerta sonora y visual.

7. Auditoría y reportes

Generación de reportes en PDF: views/generate_reporte_pdf.php. Filtros: fechas, área, estado.

Figura: Generar reporte PDF.

Auditoría

Ver registros en auditoria_sistema.php. Incluye IP, usuario, acción y timestamp.

8. Seguridad y sesiones

Acceso protegido por sesiones. Páginas como ver_tickets.php requieren sesión activa; de lo contrario redirigen a login.php.

Flujos de clave: gestion_clave.php, procesar_recuperacion.php, nueva_clave.php, guardar_clave.php.

9. Base de datos y mantenimiento

Tablas importantes: `solicitante`, `especialista`, `solicitud`, `tipo`, `marca`, `area_problema`, `auditoria_general`.

Scripts de utilidad en la carpeta scripts/ (inicializadores y validadores de esquema).
